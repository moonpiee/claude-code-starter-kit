# Claude Code Starter Kit

Welcome! This Starter Kit helps you organize every Claude Code project.

## What this kit gives you

- A structured project operating system for Claude Code
- Templates that guide thinking, not just file creation
- Practical examples that show the pattern in action
- A lightweight framework for planning, architecture, execution, and reviews

## The workflow

1. Define the project in `Goals.md`
2. Shape the system in `Architecture.md`
3. Configure the working rules in `CLAUDE.md`
4. Break work into visible tasks in `Tasks.md`
5. Use the prompt library when you need a focused AI workflow for your project
6. Capture important decisions in `Decisions.md`
7. Start Claude Code! 

>If you want deep dive into the Claude Code - see [Claude Code Mastery Guide | From Your First Prompt to Autonomous Multi-Agent Engineering](https://moonpiecreates.gumroad.com/l/hxrsgw?utm_source=starter_kit_codezip)

## File guide

- `Goals.md` — state the mission, success criteria, and constraints
- `Architecture.md` — describe the system design and deployment model
- `CLAUDE.md` — give Claude the project context it must remember
- `Tasks.md` — manage backlog, sprint work, bugs, and debt
- `Rules.md` — define coding and collaboration expectations
- `Decisions.md` — record the reasoning behind important choices
- `prompts/` — reusable prompts for planning, debugging, review, and testing
- `examples/` — real-world examples you can copy and adapt
- `screenshots/` — place visuals or reference images here

## How to use this kit

- Start by replacing the example content with your own project details. 
- Keep every section concrete. A good template should help a team make decisions quickly and reduce ambiguity.

## Recommended order

- Fill `Goals.md` first
- Draft `Architecture.md` next
- Create or update `CLAUDE.md`
- Add your first tasks to `Tasks.md`
- Review the prompt library when you need a stronger workflow

## Why this format works

Every template in this repo follows the same pattern:

- What to write
- Why it matters
- A real example

That makes the documents easier to use and easier to trust.
