# Backlog

## What to write

Capture the ideas and work items that are not yet planned for the current sprint.

## Why it matters

A backlog keeps future work visible without forcing the team to act on it immediately.

## Example

- Add export to PDF
- Support meeting recordings from Zoom and Teams
- Improve contributor onboarding docs

---

# Current Sprint

## What to write

List the work that should be completed in the near term.

## Why it matters

The current sprint should be actionable and measurable.

## Example

- Implement meeting summary generation
- Add basic search over saved summaries
- Ship the first internal beta experience

---

# Completed

## What to write

Track the work that has shipped.

## Why it matters

A completed section helps the team celebrate progress and avoid duplicate effort.

## Example

- Basic meeting upload flow
- Initial user authentication
- Prompt template library created

---

# Bugs

## What to write

Record issues that need attention.

## Why it matters

Bugs should be visible and triaged quickly.

## Example

- Long transcripts occasionally fail to process
- Search results are missing punctuation in some summaries

---

# Ideas

## What to write

Capture improvement ideas that are not yet approved.

## Why it matters

Ideas often become future priorities once the team learns more.

## Example

- Add AI-generated follow-up questions for each meeting
- Introduce team-level dashboards

---

# Technical Debt

## What to write

Record shortcuts or maintenance work that should be addressed later.

## Why it matters

Technical debt can quietly slow delivery if it is not visible.

## Example

- Replace custom caching logic with a shared library
- Refactor transcription job orchestration for clarity
