# Project Goal

Build a Next.js customer support portal that helps support teams manage tickets and customer context in one place.

# Success Criteria

- 200 active support agents in beta
- Average ticket response time reduced by 30%
- 95% of tickets have a complete customer timeline

# Constraints

- Next.js
- PostgreSQL
- Authenticated workspace
- 6-week MVP timeline

---

# Project Overview

This example project is a Next.js support portal for managing customer tickets.

# Objectives

- Give support teams a single place to review tickets
- Improve response quality with AI-assisted suggestions
- Keep customer context visible at all times

# Tech Stack

Next.js, TypeScript, PostgreSQL, and an AI completion provider.

# Coding Standards

- Prefer accessible UI patterns
- Keep components focused and reusable
- Validate data at the boundary

# Folder Structure

- `src/app` for routes and pages
- `src/components` for UI building blocks
- `src/lib` for shared logic

# Commands

- `npm run dev`
- `npm run test`

# Architecture Notes

The frontend should stay thin and delegate complex logic to the API layer.

# Current Sprint

- Ship ticket inbox and detail pages
- Add AI response suggestions

# Known Constraints

- Must support fast iteration for support workflows
- Must remain easy to navigate on desktop and tablet

# Things Claude Should Always Remember

Make the experience fast, readable, and useful for high-volume support teams.
